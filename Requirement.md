教学机构管理系统竟品分析研究

## 系统目标：

面向中小型教学培训机构，系统提供机构介绍、课程展示、课程报名购买、后台课程管理、学员管理、报名收费管理和仪表盘分析功能。系统展示机构课程信息，也能够帮助管理员完成课程、学员、报名与收费数据的统一维护。


## 系统用户分析；
系统主要包含两类用户：访客用户和管理员用户

1. 访客用户主要需求:
可以访问机构的Home page，查看教学机构的介绍，浏览课程列表、查看你课程详情、并在课程购买页面提交报名信息。

- 查看机构介绍信息（主页面）
- 查看课程列表（页面）
- 查看课程详情（页面）
- 提交报名信息（表单）
- 获得报名成功提示（等待教学机构的回复）
2. 管理员用户主要需求：
通过账号密码登录后台管理系统，维护课程信息、学员信息管理、报名收费管理和仪表盘数据查看。

- 登录后台管理系统（Login页面）
- 维护课程信息
- 维护学员信息
- 查看和处理报名记录
- 录入缴费金额和优惠金额
- 查看系统统计数据

（可以扩展教师端、学员端）

## 功能性需求

1. 首页展示模块：

**功能说明**：

首页用于展示教学机构的基本信息和核心卖点，是访客进入系统后的第一入口。

**具体功能**：

1. 展示机构名称、宣传语和简介。
2. 展示机构优势，例如师资专业、课程系统、价格透明、报名便捷。
3. 展示热门课程或推荐课程。
4. 提供课程列表入口。
5. 提供后台登录入口。
6. 页面内容建议

**首页可以包括：**

顶部导航栏
机构介绍区域
核心卖点区域
热门课程区域
报名引导按钮
页脚信息

2. 课程展示模块；

**功能说明**:

课程展示模块面向访客开放，用于展示机构开设的课程信息。

**具体功能**:
1. 查看课程列表。
2. 查看课程详情。
3. 按课程分类浏览课程。
4. 查看课程价格、课时、教师、适合人群等信息。
5. 点击“立即报名”进入报名表单页面。

**课程信息包括**:
- 课程名称
- 课程分类
- 课程价格
- 课程课时
- 授课教师
- 课程简介

课程状态建议设置为：

- 上架
- 下架

只有上架课程会展示给访客。

3. 访客报名模块

**功能说明**：

访客报名模块是系统的核心转化功能。访客不需要注册账号，只需要填写基本信息即可提交报名意向。

**具体功能**：

1. 访客选择课程后进入报名页面。
2. 访客填写姓名、手机号、备注信息。
3. 系统自动记录报名课程、课程价格、报名时间。
4. 报名记录初始状态为“待联系”。
5. 提交成功后显示报名成功提示。
6. 报名状态设计

**报名状态建议设置为**：

状态	说明
待联系	访客刚提交报名，管理员还未处理
已联系	管理员已经联系过访客
已报名	访客确认报名
已取消	访客放弃报名

4. 管理员登录模块

**功能说明**：

管理员登录模块用于保护后台管理功能。

**具体功能**：
1. 管理员输入账号和密码登录后台。
2. 登录成功后进入管理员仪表盘。
3. 登录失败时显示错误提示。
4. 未登录用户不能访问后台页面。
5. 管理员可以退出登录。
6. 简化设计

系统只保留管理员账号，不做普通用户注册。

5. 课程管理模块

**功能说明**

课程管理模块用于管理员维护前台展示的课程信息。

**具体功能**：
1. 新增课程。
2. 修改课程。
3. 删除课程。
4. 查询课程。
5. 设置课程上架或下架。
6. 查看每门课程的报名数量。

**设计重点**

课程管理模块是后台的基础模块，管理员维护的课程信息会直接影响前台展示内容。

6. 报名管理模块

**功能说明**

报名管理模块用于管理员处理访客提交的报名信息。

**具体功能**

1. 查看全部报名记录。
2. 按课程名称查询报名记录。
3. 按报名状态查询报名记录。
4. 按手机号查询报名记录。
5. 修改报名状态。
6. 删除无效报名记录。
7. 查看报名详情。

**报名记录信息包括**

- 报名编号
- 访客姓名
- 手机号
- 意向课程
- 课程价格
- 报名时间
- 报名状态
- 缴费状态
- 备注信息


7. 报名费用财务模块

**功能说明**

财务模块用于管理员维护报名费用和缴费状态，是系统区别于普通课程展示网站的重点功能。

**具体功能**
1. 报名记录生成后，系统自动记录课程费用。
2. 管理员可修改缴费状态。
3. 管理员可查看已缴费金额。
4. 管理员可查看待缴费金额。
5. 管理员可按课程统计收入。
6. 管理员可按时间统计收入。

**缴费状态设计**
状态	说明
未缴费	访客提交报名后默认状态
已缴费	管理员确认收到费用

8. 管理员仪表盘模块

**功能说明**

仪表盘用于展示系统核心运营数据，是后台首页。

**具体功能**
1. 显示课程总数。
2. 显示报名总数。
3. 显示待联系人数。
4. 显示已报名人数。
5. 显示已缴费人数。
6. 显示总收入。
7. 显示热门课程排行。
8. 显示课程收入排行。

**仪表盘展示方式**

数据卡片(先使用图片)

## 系统模块结构

```
教学机构课程管理系统
│
├── 前台访客端
│   ├── 首页介绍
│   ├── 课程列表
│   ├── 课程详情
│   └── 访客报名
│
└── 后台管理员端
    ├── 管理员登录
    ├── 仪表盘
    ├── 课程管理
    ├── 报名管理
    └── 财务管理
```


## 非功能性需求

1. 易用性需求
- 访客无需注册即可报名。
- 课程信息展示清晰。
- 报名表单字段尽量少。
- 管理员后台菜单结构明确。
- 管理员修改状态后应有成功提示。

2. 安全性需求
- 后台管理页面必须登录后才能访问。
- 管理员密码不应在页面中明文展示。
- 访客不能访问后台接口。
- 用户输入内容需要进行基本校验。
- 手机号不能为空，课程编号不能为空。

3. 性能需求
- 首页、课程列表和课程详情页面应快速加载。
- 报名记录较多时，后台应支持分页查询。
- 仪表盘统计数据应控制查询复杂度。

4. 可维护性需求
- 系统采用分层架构设计。
- 前台展示、后台管理、业务逻辑和数据库操作分离。
- 报名管理和财务管理可以共用报名记录数据。
- 课程模块应便于后续扩展教师、班级、课表等功能。

5. 可扩展性需求

后续可以扩展：

学生账号系统
教师管理。
班级管理。
在线支付。
优惠活动。
学员学习记录。
工作人员联系。


本系统采用 MVC 架构与分层架构相结合的模块化单体设计。系统整体划分为视图层、控制层、业务逻辑层、数据访问层和数据库层。视图层由 Vue 页面组成，负责数据展示和用户操作；控制层由 Spring Boot Controller 组成，负责接收前端请求并返回统一响应；业务逻辑层由 Service 组成，负责处理学员报名、缴费状态计算、重复报名校验等核心业务；数据访问层由 Mapper 组成，负责与 MySQL 数据库交互。
